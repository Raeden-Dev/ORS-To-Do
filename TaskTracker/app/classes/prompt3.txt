Phase 1: Data Model & Migration Foundation (The Engine Room)
Currently, your tasks are linked to an enum called OriginModule (QUICK, DAILY, WORK). We cannot use hardcoded enums for dynamic sections.

Create SectionConfig: We will build a new object inside AppStats that stores a section's name, ID, and a massive list of boolean toggles (e.g., hasStreak, showAnalytics, enableSubTasks, showPriority, etc.).

Update TaskItem: We will change OriginModule to a dynamic String sectionId.

Safe Data Migration: We will write a silent startup script that converts your old .dat files. It will auto-generate 3 dynamic sections (Quick, Daily, Work) configured to act exactly like the old ones, and map your existing tasks to them so you lose no data.

Phase 2: The Universal DynamicModuleFX (The Chameleon UI)
Currently, you have two separate files (TaskModuleFX and DailyModuleFX). We will combine them into one ultimate class.

Conditional Rendering: We will build DynamicModuleFX.java. When it loads, it will read the SectionConfig.

If config.hasPriority() is true, it renders the Priority Box and Side Rectangle. If false, it hides them.

If config.hasStreak() is true, it renders the midnight countdown timer and fire emoji at the top.

If config.showTags() is true, it generates the filter pills.

We will wire up all the required toggles you requested so one single Java class can morph into any type of list you want.

Phase 3: Dynamic App Routing & Sidebar
Currently, TaskTrackerFXApp.java hardcodes the 5 buttons on the left.

Dynamic Sidebar: We will change the sidebar to loop through your AppStats.getSections() list and generate the buttons on the fly.

Dynamic Routing: When you click a sidebar button, instead of pulling from a hardcoded variable, it will instantiate a new DynamicModuleFX passing in that specific section's configuration.

Phase 4: Settings Panel Overhaul (The Control Center)
We will add a powerful new "Section Manager" to SettingsModuleFX.java.

Section Builder: A UI where you can click "Create New Section".

Toggle Matrix: It will open a dialog with checkboxes for all your requested features (Show Date, Allow Archive, Track Time, Streak System, etc.).

Live Updates: Saving the changes will instantly regenerate the Sidebar and apply the new rules to the tasks inside.

Phase 5: Logic Cleanups (Analytics & Archiving)
Dynamic Analytics: We will rewrite the HTML export engine so it dynamically handles whatever section it is triggered from, rather than being hardcoded to "WORK" or "DAILY".

Archive Panel: We will update the Archive tab so that it labels tasks based on their dynamic section name instead of the old enum.
