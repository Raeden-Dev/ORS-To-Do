1. The "Reward Shop" (Gamification 2.0)
Right now, you are accumulating a massive Global Score, but the points just sit there as a high score.
The Feature: Create a new dynamic page called "Reward Shop." You can create "Reward Cards" that cost points. For example, "Play 1 Hour of Hytale" costs 500 points, or "Watch YouTube guilt-free" costs 200 points.
How it works: When you click a reward, it deducts the points from your globalScore and adds a log to your Analytics. This closes the gamification loop by giving you a real, tangible reason to farm task points!

2. Zen Mode
When a list has 20 (Make this configurable in General Configuration settings) tasks on it, it can cause "task paralysis" where you end up doing nothing.
The Feature: A button at the top of the Dynamic Module that toggles "Zen Mode."
How it works: It applies a CSS overlay that completely hides the sidebar, the headers, and all tasks except for the single highest-priority, unlocked task on your list. It forces your screen to just show ONE thing to do, completely eliminating distractions.
If there is no priority present in the page, it'll randomly select a task

3. Global Search (Heavy QoL)
As your database grows and tasks get archived, finding a specific note or sub-task becomes impossible.
The Feature: A global search bar at the very top of your sidebar (above the Navigation buttons).
How it works: Typing in it replaces the center BorderPane with a temporary "Search Results" module that scans every task's text, sub-tasks, and work types across all sections (even the archive) and displays the matched cards.

4. Sub-Task Dependencies
You currently have dependencies linking entirely separate task cards.
The Feature: Make the dependency lock apply within a single task card.
How it works: If a task has 5 sub-tasks, the main checkbox for the entire card is completely locked/disabled until all 5 sub-tasks are checked off. This forces you to respect the micro-steps of complex development tasks.
